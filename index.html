Help & Tutorial

Getting Started

  1. Create Entities: Drag entity types from the left palette onto the canvas.
  2. Connect Entities: Click on a source entity, then click on a target entity to create a transaction.
  3. Edit Properties: Click on any entity to view and edit its properties in the right panel.
  4. Analyze: Click the "Analyze" button to calculate risk scores and detect patterns.
  5. Playback: Use the timeline controls at the bottom to animate transaction flows.

Entity Types

  • Originator: Source of illicit funds (Placement stage)
  • Shell Company: Company with no real operations (Layering stage)
  • Correspondent Bank: Bank facilitating international transfers
  • Nominee Account: Account held in another person's name (Layering stage)
  • Crypto Exchange: Cryptocurrency conversion (Layering stage)
  • Cash Business: Cash-intensive business (Integration stage)
  • Final Beneficiary: Ultimate recipient (Integration stage)

Pattern Detection

The simulator can automatically detect common money laundering patterns:

  • Smurfing: Multiple small transactions to avoid reporting thresholds
  • Round-Tripping: Funds returning to origin after layering
  • Rapid Movement: Quick succession of transfers through entities
  • Layering Complexity: Complex networks with many intermediaries

📖 About the AML Nesting Flow Simulator

The AML Nesting Flow Simulator is an interactive, browser-based tool for visualising, constructing, and analysing money laundering scenarios. It enables analysts, compliance officers, investigators, and trainees to build financial crime flow diagrams, detect suspicious patterns automatically, and score entity risk — all without any backend or installation required.

Canvas & Entity Management

  • 17 entity types available in the left palette: Originator, Shell Company, Correspondent Bank, Nominee Account, Crypto Exchange, Cash Business, Final Beneficiary, Trade Company, Real Estate, Offshore Trust, Money Service Business, Professional Service Provider, Casino, Precious Metals Dealer, Investment Fund, Merchant, and NPM FINTECH
  • Drag-and-drop entities from the palette onto the canvas, or double-click to place them at the centre
  • Freely reposition entities by dragging them anywhere on the 5 000 × 5 000 virtual canvas
  • Pan the canvas by clicking and dragging empty space; zoom in and out with the toolbar buttons
  • Each entity node displays its icon, name, type, jurisdiction, AML stage badge, and risk score

Transaction Lines

  • Click-to-connect: click entity A then entity B to draw a directed payment line with an arrow
  • Lines use smart auto-routing with continuous anchors to avoid overlap
  • Each line shows a currency denomination label at its midpoint
  • Lines are colour-coded by flags and method:
    • 🔵 Blue — cross-border transaction
    • 🟣 Purple dashed — on-behalf transaction
    • 🔷 Dark blue dashed — both cross-border and on-behalf
    • 🔴 Red dashed — suspicious (overrides other colours)
  • Click a line to select it and view or edit its properties in the right panel

Property Panel

Entity properties: name, type, jurisdiction, AML stage, notes, risk score meter with factor breakdown, and Connect / Save / Delete actions.

Transaction properties: currency, payment method (Wire Transfer, Cash, Crypto, Check, Trade Invoice), description, cross-border flag, on-behalf flag with entity dropdown, and Save / Delete actions.

Risk Analysis

  • Press Analyze to compute an overall risk score (0–100) rated Low / Medium / High / Critical
  • The Top 5 riskiest entities are ranked with their score and top contributing factors
  • Risk factors include entity type base risk, jurisdiction risk, transaction volume, suspicious flags, and AML stage weighting

Pattern Detection

Eight AML patterns are detected automatically after running Analyze:

  • 🐢 Smurfing — multiple sub-threshold transactions from one source
  • 🔄 Round-Tripping — funds that circulate back to the originator via a cycle
  • ⚡ Rapid Movement Chain — 3+ linked transfers within a short time window
  • 🕸️ Complex Layering Network — 5+ entities across multiple jurisdictions with heavy shell company use
  • 🚢 Trade-Based Money Laundering — import/export entities with suspicious invoice flows
  • 🐦 Cuckoo Smurfing — 3+ sources depositing sub-threshold amounts to a single target
  • 🌀 Cryptocurrency Mixing — crypto exchanges with balanced many-in / many-out flows
  • 🪆 Nesting Risk — 2 or more connected correspondent banks, NPM FINTECHs, shell companies, nominee accounts, or offshore trusts forming a layering chain before integration

Each detected pattern shows a severity badge, a plain-language description, a Highlight button (illuminates involved nodes and lines for 4 seconds), and a Learn More button with full educational content including indicators, real-world examples, red flags, prevention, and legal consequences.

Simulation Management

  • New — start a fresh simulation
  • Save / Load — persist simulations to browser localStorage and reload them by name
  • Export JSON — download the full simulation as a JSON file
  • Export PDF — export the canvas view as a PDF report
  • Import — load a previously exported JSON file

Timeline Player

  • Animate transaction flows step-by-step along the canvas using Play, Pause, Stop, Step Back, and Step Forward controls
  • Adjustable playback speed (0.5×, 1×, 2×, 4×)
  • A scrubber bar for navigating to any point in the transaction sequence
